## 需求理解（按你描述）
- 双击 Ctrl：负责“换出/隐藏”界面，并且唤出时保持上一次的窗口形态（上次是完整→唤出完整；上次是迷你→唤出迷你）。
- 移除现有“单键唤醒/组合键唤醒”这一套（目前 config 里有 GlobalHotkey/SingleHotkey 两个入口，最终都 ToggleMainWindow）。
- 新增两组独立热键：
  - 完整窗口热键：可设置为组合键或单键（例如 Alt+W 或 F2）。按下后：若窗口隐藏→直接唤出完整；若当前显示迷你→切到完整；若当前已是完整→再按一次隐藏。
  - 迷你窗口热键：同理（例如 Alt+Q 或 F3）。按下后：若窗口隐藏→直接唤出迷你；若当前显示完整→切到迷你；若当前已是迷你→再按一次隐藏。
- 检查并避免项目中其他硬编码热键冲突。

## 现状调研结论（与你的需求对照）
- 双击 Ctrl 已存在并调用 ToggleMainWindow，ToggleMainWindow 会记录/恢复上一次 IsFullMode，因此基本符合“保持上次窗口形态”。
- 现有 HotkeyManager 注册点只有一处：MainViewModel.UpdateGlobalHotkey() 注册 ToggleWindow(组合键) + ToggleWindowSingle(单键)，默认 Alt+Space（可能与系统菜单冲突）。
- 除此之外还有 GlobalKeyService 的“双击分号/符号序列”监听（属于字符序列唤醒），需要纳入冲突检查范围。

## 实现方案（分步）
### Step 1：配置模型调整（AppConfig）
- 删除/弃用：GlobalHotkey、SingleHotkey（以及相关 UI）。
- 新增：
  - FullWindowHotkey（string，例如 "Alt+W" 或 "F2"）
  - MiniWindowHotkey（string，例如 "Alt+Q" 或 "F3"）
- 迁移策略（避免老用户丢配置）：
  - 若 FullWindowHotkey 为空且 GlobalHotkey 非空：FullWindowHotkey = GlobalHotkey。
  - 若 MiniWindowHotkey 为空且 SingleHotkey 非空：MiniWindowHotkey = SingleHotkey（或保持空并提示用户重新设置；我会在实现时选更安全的一种并写在变更说明里）。

### Step 2：设置页 UI 重做（快捷键窗口区块）
- 移除：单键唤醒、组合键唤醒的两行输入框。
- 新增两行：
  - 完整窗口｜右侧：热键输入框（一次录入同时支持单键/组合键）
  - 迷你窗口｜右侧：热键输入框（同上）
- 若现有输入控件无法同时录入单键与组合键：在每行前增加“单键/组合键”勾选项，并基于勾选启用对应录入规则。

### Step 3：热键解析与注册重构（替换 UpdateGlobalHotkey）
- 新增方法：UpdateWindowHotkeys()
  - 注册：ToggleFullWindowHotkey、ToggleMiniWindowHotkey（两个独立 ID）
  - 解析规则沿用现有字符串格式："Ctrl+Alt+Shift+Win+Key" 或仅 "F2"（无 modifiers）。
- 清理旧注册：启动时 Remove("ToggleWindow")/Remove("ToggleWindowSingle")，确保不再响应旧入口。

### Step 4：新增“按目标模式切换”的显隐逻辑
- 新增方法 ToggleWindowToMode(targetFull: bool)：
  - 若窗口隐藏：设置 IsFullMode=targetFull → Show/Activate/Focus。
  - 若窗口显示且当前模式!=targetFull：切换 IsFullMode=targetFull 并保持显示。
  - 若窗口显示且当前模式==targetFull：Hide()。
- 双击 Ctrl：继续使用 ToggleMainWindow（保持“上次形态”逻辑）。

### Step 5：冲突检查与防呆
- 保存设置或注册热键前做校验：
  - FullWindowHotkey 与 MiniWindowHotkey 不允许相同；相同则拒绝保存或自动清空其中一个并提示。
  - 与“双击 Ctrl/符号序列唤醒”不属于同一机制，但会提醒用户避免把热键设成常用输入字符导致误触。
  - 默认 Alt+Space 建议改掉（与系统菜单常冲突）。

### Step 6：全局扫描硬编码热键
- 仅一处 HotkeyManager 注册点 + AppConfig 默认值；其余为 GlobalKeyService 的双击/序列监听。
- 实现时会再做一次全库搜索，确认没有其他 AddOrReplace/硬编码键位。

### Step 7：验证
- Release 构建。
- 手动回归：
  - 双击 Ctrl：从完整/迷你各自隐藏后再唤出，形态保持正确。
  - 完整热键：隐藏→唤出完整；迷你显示时→切完整；完整显示时→隐藏。
  - 迷你热键：隐藏→唤出迷你；完整显示时→切迷你；迷你显示时→隐藏。
  - 同键冲突时的提示/处理。
