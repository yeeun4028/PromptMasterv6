# 现代设置界面 UI 重构与设计方案

作为 UI 设计师，我为您制定了以下基于 `#F1F1EF` 背景色的现代极简风格设计方案。我们将把设置界面从主窗口中解耦，提升代码可维护性和界面美观度。

## 1. 设计规范 (Design System)
*   **色彩体系**:
    *   **主背景**: `#F1F1EF` (暖灰/米色，营造纸质般的舒适感)
    *   **卡片背景**: `#FFFFFF` (纯白，用于承载内容，带圆角)
    *   **主色调 (Accent)**: `#4183C4` (保持现有品牌蓝，用于主按钮/选中状态)
    *   **文本**: `#333333` (主标题), `#666666` (次级文本), `#999999` (提示文本)
*   **视觉语言**:
    *   **圆角 (Radius)**: 窗体 `12px`，卡片 `8px`，按钮 `6px`。
    *   **阴影 (Shadow)**: 使用低透明度的大扩散阴影 (`DropShadowEffect`) 营造悬浮感。
    *   **布局**: 左侧导航 + 右侧内容。内容区采用“卡片式”分组，将表单项包裹在白色卡片中。

## 2. 架构调整 (Refactoring)
*   **模块化**: 创建 `Views` 文件夹，新建 `SettingsView.xaml` 用户控件。
*   **解耦**: 将 `MainWindow.xaml` 中原本 500+ 行的设置代码移除，替换为 `<views:SettingsView />`。

## 3. 分步实施计划 (Implementation Steps)

### Phase 1: 基础架构与样式定义
1.  创建 `Views/SettingsView.xaml`。
2.  定义资源字典：包含新的配色画笔 (Brushes) 和基础控件样式 (CardStyle, ModernButtonStyle, ModernTextBoxStyle)。
3.  搭建主框架：实现 600x450 的弹窗容器，应用 `#F1F1EF` 背景和左侧导航栏布局。

### Phase 2: 内容迁移与重设计
1.  **左侧导航栏**: 重新设计 Tab 按钮，移除旧版边框，采用背景色块或左侧指示条表示选中状态。
2.  **内容区域 (右侧)**: 依次迁移并美化 5 个功能模块：
    *   **快捷键**: 使用白色卡片包裹设置项。
    *   **自动化**: 优化 RadioButton 布局，增加呼吸感。
    *   **AI模型**: 重绘模型列表项，使其更像现代列表控件。
    *   **同步 & 极简窗口**: 统一输入框风格，移除默认边框，使用底部线条或平坦填充。

### Phase 3: 集成与验证
1.  在 `MainWindow.xaml` 中引用新控件。
2.  绑定 ViewModel 数据，确保所有功能（保存、切换 Tab）正常工作。
3.  微调视觉细节（间距、字体大小）。
