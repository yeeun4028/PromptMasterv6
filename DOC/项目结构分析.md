# PromptMasterv6 项目结构分析报告

## 1. 核心架构理念：垂直切片 (Vertical Slice Architecture)

项目采用了 **Jimmy Bogard** 提倡的垂直切片架构。与传统的层级架构（UI -> Business -> Data）不同，本项目按**功能 (Feature)** 组织代码。

### 核心原则
- **功能自包含**：每个功能模块（如 `Settings`, `Launcher`）拥有自己的 View、ViewModel 和相关的逻辑代码。
- **按功能解耦**：利用 `MediatR` 进行功能间的通信，减少强耦合。
- **高内聚**：相关的逻辑物理上放在一起，易于维护和扩展。

## 2. 物理文件夹结构分析

| 目录 | 职责描述 |
|:---|:---|
| `Features/` | **核心业务功能区**。包含垂直切片后的各个功能模块。 |
| ├── `Main/` | 主窗口界面 (`MainWindow`)、主 ViewModel (`MainViewModel`) 及 LaunchBar。 |
| ├── `Workspace/` | 提示词工作区逻辑，处理文件列表和内容编辑。 |
| ├── `Launcher/` | 全局启动器功能，支持搜索和快速执行。 |
| ├── `Settings/` | 设置中心，已拆分为 `AiModels`, `ApiProviders`, `Sync` 等子模块。 |
| ├── `ExternalTools/` | 外部工具集成，如 OCR、翻译、截图等。 |
| ├── `PinToScreen/` | 贴图/置顶显示功能。 |
| └── `Shared/` | 功能间共享的逻辑，包括 `VariableService`, `ContentConverterService` 等。 |
| `Core/` | **核心抽象层**。包含基础接口 (`Interfaces`)、模型 (`Models`) 和消息定义 (`Messages`)。 |
| `Infrastructure/` | **基础设施层**。包含各种具体服务实现（AI、文件、WebDAV、日志、热键、窗口管理等）。 |
| `Themes/` & `Resources/` | **UI 资源**。包含全局样式、HandyControl 扩展、图标和动画资源。 |
| `DOC/` | **项目文档**。记录架构重构过程、业务逻辑分析和进展。 |

## 3. 关键技术栈

- **框架**: .NET 8.0-windows (WPF)
- **MVVM**: `CommunityToolkit.Mvvm` (由 Microsoft 维护，提供高效的 ObservableObject 和 RelayCommand)。
- **中介者模式**: `MediatR` (用于特征间的松耦合通信和请求处理)。
- **依赖注入 (DI)**: `Microsoft.Extensions.DependencyInjection` (在 `App.xaml.cs` 中配置)。
- **UI 库**: `HandyControl` (提供丰富的现代 WPF 控件)。
- **AI 对接**: `Betalgo.OpenAI` 及自定义服务（支持 OpenAI 兼容接口、百度、腾讯、谷歌等）。
- **交互增强**: `NHotkey.Wpf` (全局热键)、`gong-wpf-dragdrop` (拖拽支持)、`Markdig.Wpf` (Markdown 渲染)。

## 4. 核心流程与机制

### 4.1 程序启动流程 (Bootstrapping)
- 入口点：`App.xaml.cs` 的 `OnStartup` 方法。
- 初始化内容：
  1. 创建 `ServiceCollection` 并调用 `ConfigureServices` 注册所有服务和 ViewModel。
  2. 构建 `ServiceProvider` 并存储在 `App` 实例中。
  3. 通过 `WindowRegistry` 注册视图与 ViewModel 的映射关系。
  4. 显示 `MainWindow` 和 `LaunchBarWindow`。
  5. 启动 `GlobalShortcutCoordinator`。

### 4.2 数据存储与同步
- **抽象**: `IDataService` 接口定义了数据操作。
- **实现**: 
  - `FileDataService`: 本地文件存储。
  - `WebDavDataService`: 自定义 WebDAV 同步存储。
- **机制**: 在 `App.xaml.cs` 中根据键值（"cloud" 或 "local"）进行 Keyed DI 注入，支持无缝切换。

### 4.3 AI 功能路由
- `AiService`: 作为中枢神经，根据配置选择对应的云端服务商（Baidu, Google, Tencent, OpenAI 等）。
- 兼容性：支持多模型适配（如 GPT-4, Claude, Whisper 等）。

## 5. 架构演进说明

根据 `DOC` 目录下的重构记录，项目刚经历了一次从“巨型 ViewModel”向“垂直切片”的重大迁移：
- `MainViewModel` 和 `SettingsViewModel` 的职责被大幅剥离。
- 业务逻辑被提取到 `Features/Shared/Services`。
- 窗口管理由 `WindowManager` 和 `WindowRegistry` 统一接管，实现了更好的测试性和可维护性。
