# 业务逻辑分析：添加提示词

## 1. 核心流程概述
添加提示词的逻辑主要分布在 `SidebarViewModel` (创建)、`MainViewModel` (导入与管理) 以及底层的数据服务中。

## 2. 详细逻辑步骤

### A. 通过侧边栏创建 (常规方式)
1. **触发命令**：用户点击“新建文件”按钮，执行 `SidebarViewModel.CreateFileCommand`。
2. **实例化**：创建一个新的 `PromptItem` 对象。
    - `Title` 初始化为空字符串。
    - `Content` 初始化为空字符串。
    - `FolderId` 绑定到当前选中的文件夹 ID。
    - `LastModified` 设置为当前时间。
3. **加入集合**：将该对象添加到 `Files` 的 `ObservableCollection` 中。由于 UI 绑定了该集合，侧边栏会立即显示新项。
4. **UI 交互**：通过 `WeakReferenceMessenger` 发送 `RequestSelectFileMessage`，使 UI 自动选中新创建的提示词并进入编辑模式。
5. **触发保存**：发送 `RequestSaveMessage` 消息。

### B. 通过 Markdown 导入
1. **触发命令**：`MainViewModel.ImportMarkdownFilesCommand`。
2. **文件选择**：弹出文件对话框，允许用户多选 `.md` 或 `.markdown` 文件。
3. **批量创建**：遍历选中的文件路径，读取内容和标题，为每个文件创建一个 `PromptItem` 并加入 `Files` 集合。
4. **视图更新**：刷新 `FilesView` 过滤器并触发保存。

### C. 持久化逻辑
1. **自动保存监听**：`MainViewModel` 会监听 `Files` 集合的变化。当有新项目加入时，会为其附加 `PropertyChanged` 监听器。
2. **脏数据标记**：一旦发生添加、删除或内容修改，`RequestSave()` 被调用，将 `IsDirty` 设为 `true`。
3. **节流保存**：使用 Reactive (Rx) 的 `Throttle` 操作符（5秒延迟）。
4. **本地备份**：`PerformLocalBackup()` 通过 `FileDataService` 将 `Folders` 和 `Files` 序列化为 JSON 并保存到本地。
5. **云端同步**：若配置了 WebDAV，可触发 `WebDavDataService` 将数据上传到远程服务器。

## 3. 技术实现要点
- **MVVM**：利用 `CommunityToolkit.Mvvm` 进行命令绑定和属性通知。
- **消息总线**：利用 `WeakReferenceMessenger` 实现 ViewModel 间的解耦通信（如：侧边栏通知主视图保存或选择文件）。
- **响应式编程**：利用 `System.Reactive` 处理保存逻辑的节流（Throttle），避免频繁磁盘 I/O。
